Les Manipulations C (Code source)

Application : Bouton Poussoir Maintenu

Objectif : Allumer la LED uniquement lorsque le bouton est enfoncé. Utilisation de la lecture directe d'état logique.

// Extrait du while(1) - main.c
if (HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13) == GPIO_PIN_RESET)
{
    // Bouton appuyé (RESET car Pull-up) = On allume la LED
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET);
}
else
{
    // Bouton relâché = On éteint la LED
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET);
}

Test : Clignotement TogglePin

Objectif : Faire clignoter la LED avec un cycle de 500ms en utilisant la fonction d'inversion d'état.

// Extrait du while(1) - main.c
while (1)
{
    // On inverse l'état actuel de la broche PA5
    HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
    
    // Pause bloquante de 500 millisecondes
    HAL_Delay(500);
}

Le Défi : Variateur de Vitesse (Pro Level)

Objectif : Changer la vitesse (1000ms, 200ms, Fixe) à chaque appui. Solution implémentée : Machine à états non-bloquante avec HAL_GetTick() pour garantir la détection instantanée du bouton.

// Extrait du while(1) - main.c
uint32_t temps_actuel = HAL_GetTick();
uint8_t etat_bouton_actuel = HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13);

// 1. DÉTECTION DU BOUTON (Anti-rebond)
if (etat_bouton_actuel == GPIO_PIN_RESET && etat_bouton_precedent == GPIO_PIN_SET) {
    HAL_Delay(50); 
    if (HAL_GPIO_ReadPin(GPIOC, GPIO_PIN_13) == GPIO_PIN_RESET) {
        mode++; 
        if (mode > 2) mode = 0; 
        if (mode == 2) HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET); 
    }
}
etat_bouton_precedent = etat_bouton_actuel;

// 2. GESTION DE LA LED (Non-bloquant)
if (mode == 0) { // Lent
    if (temps_actuel - dernier_temps >= 1000) {
        dernier_temps = temps_actuel;
        HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
    }
} else if (mode == 1) { // Rapide
    if (temps_actuel - dernier_temps >= 200) {
        dernier_temps = temps_actuel;
        HAL_GPIO_TogglePin(GPIOA, GPIO_PIN_5);
    }
}

Questions Post-Lab (Cahier des charges)

La broche PA5 doit agir comme une source de tension/courant pour alimenter la LED (Output). Une broche configurée en entrée (Input) sert uniquement à lire un état électrique extérieur, elle ne peut pas fournir l'énergie nécessaire pour allumer un composant.

Elle force la broche à un état logique HAUT (3.3V / SET) lorsque le bouton n'est pas appuyé. Sans cela, la broche serait "flottante" et capterait des bruits électromagnétiques, provoquant des lectures aléatoires (faux appuis).

WritePin() force la broche à un état absolu et précis (soit 0, soit 1). TogglePin() lit l'état actuel de la broche et l'inverse automatiquement (si c'est 0 ça devient 1, et inversement).

Cette fonction est vitale : elle initialise la couche d'abstraction matérielle (HAL), configure le Flash prefetch, et surtout, elle démarre le timer système (SysTick) qui est indispensable pour utiliser les fonctions de délai comme HAL_Delay() et HAL_GetTick().

Par défaut, pour économiser l'énergie, les horloges des périphériques STM32 sont éteintes. Si on oublie cette macro, le port A n'est pas alimenté. L'écriture dans ses registres de configuration sera ignorée par le processeur, et la LED ne s'allumera pas.

Lorsqu'on appuie sur un bouton physique, les contacts métalliques à l'intérieur vibrent ou "rebondissent" microscopiquement avant de se stabiliser. Le microcontrôleur est si rapide qu'il interprète ces vibrations comme plusieurs dizaines d'appuis très rapides au lieu d'un seul.

Non, c'est dangereux. Bien que les broches aient des résistances internes (pull-up/pull-down), elles sont conçues pour des signaux logiques (faible courant, ~40kΩ) et rendraient la LED invisible. Configurée en sortie standard, la broche délivrera trop de courant, ce qui risque de griller la LED ou, pire, d'endommager le port du microcontrôleur. Une résistance externe est obligatoire.

Le microcontrôleur est totalement "bloqué" (gelé) pendant ce délai. Tout appui sur le bouton relâché avant la fin de la seconde sera ignoré. Le changement n'est absolument pas instantané.

Il faut abandonner HAL_Delay(). La solution professionnelle (utilisée dans notre code du défi) est de créer une Machine à états non-bloquante utilisant HAL_GetTick() pour mesurer le temps écoulé sans geler le CPU. Une autre excellente solution matérielle consiste à utiliser les Interruptions externes (EXTI).
